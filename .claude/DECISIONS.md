# Decisions

Decisions made during the frontend API integration (prompt_2.md).

## Field visibility in forms

| Field | Decision |
|---|---|
| `id` | Auto-generated — never shown in create/edit forms |
| `sku` | Auto-generated by backend — hidden from create form; visible in tables and detail views |
| `estoque_atual` | Managed exclusively by stock movement endpoints — not editable directly |
| `estoque_baixo` / `sem_estoque` | Computed properties on the backend — display only |
| `criado_em` / `atualizado_em` | Auto-generated timestamps — shown only in tables, not in forms |

## Endpoint mapping (SaidaEstoque)

The old `saidaAPI` called non-existent routes. Mapped to real backend routes:

| UI type button | `tipo_movimentacao` sent to backend |
|---|---|
| Venda | `SAIDA_VENDA` |
| Saída | `SAIDA_BAIXA` |
| Perda | `SAIDA_BAIXA` |

## `responsavel` field in SaidaEstoque

The backend `SaidaEstoqueCriar` has no `responsavel` field. The form field was kept
in the UI (same layout) but is now mapped to `documento_referencia` in the payload.
In the history table, the "Responsável" column shows `documento_referencia`.

## Dashboard "4th card" metric

The original design showed "Vendas do Mês". The backend exposes only daily metrics
(`movimentacoes_hoje`, `saidas_hoje`). The card was renamed to "Movimentações Hoje"
and shows `movimentacoes_hoje` with a breakdown of entries vs exits.

## `toggle-ativo` for suppliers

No `PATCH /fornecedores/{id}/toggle-ativo` route exists. Replaced with
`PUT /fornecedores/{id}` sending `{ ativo: !current_ativo }` which achieves the
same result using the existing update endpoint.
